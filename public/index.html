What and why?

Blog template to display records with graphical and textual information in elegant and non-dull manner.

Gallery demonstrates a bunch of JS and CSS approaches to provide a good user experience.

Gallery and slides use plain Javascript, with no frameworks and libraries (except jQuery for AJAX calls)

Backend API on top of PostgreSQL DB added for demonstration purposes

Gallery layout

Layout should look like Masonry, BUT... instead of placing elements in optimal position based on available vertical space we display items in chronological order and fit them to strict positioned boxes

Than we move containers to make appearance not such ordered, for example: item 6 moved to left and block of two horizontal boxes - to center

Gallery looks nice on any screen size when have to display number of records multiple of 15.

Check next slide to learn WHAT IF...

Q:If we have number of records that break the harmony?

A:It's simple, just put them away in extra row with different display rules.

Decision about extra row is based on number of items to display and viewport width.

Presentation layer solutions

  • ✔Javascript to set and change classes to control visual appearance
  • ✔CSS Grid and media-queries to design responsive layout
  • ✔CSS transform property to reorder items
  • ✔SASS variables and mixins to define transform: translate value based on block dimensions and margins

What is on back-end

Back-end tools

  • ✔Koa.js for RESTful API
  • ✔Knex.js for DB management and interaction
  • ✔Chai for TDD
  • ✔grunt-connect-proxy to avoid CORS issues
  • ✔PostgreSQL as database
